Wave 0

Foundation Layer — "Build Before You Deconstruct"

Establish observability, IaC baseline, AWS Landing Zone, and re-org team topology before touching the monolith. You cannot safely decompose a system you cannot observe.
✓ COMPLETE Q1–Q2 2025 · 6 months
Strategic Focus
  • Deploy OpenTelemetry collectors across all monolith services — establish distributed traces before first cut.
  • Adopt Terraform Enterprise for IaC across all 6 AWS accounts. No manual console changes.
  • Implement AWS Landing Zone with service control policies (SCPs) and guardrails per team account.
  • Restructure org from functional to domain-aligned teams (Inverse Conway Maneuver).
  • Launch Backstage IDP with service catalog for all existing monolith modules.
60PB Data Plane Levers
  • S3 Object Storage Lens enabled — baseline all 60PB for class, age distribution, and access patterns.
  • Activate S3 Intelligent-Tiering on data between 30–730 days (18 PB identified). Target $2.1M/year savings.
  • Object tagging standard enforced: tenant-id, data-class, retention-epoch applied to all new objects.
  • Establish daily COGS dashboard. Baseline: $24,800/day as the line we improve from.
Tech Debt Strategy
  • Debt Inventory: Catalog all monolith coupling points via ArchUnit analysis and static dependency graphs.
  • Critical path: identify that Elasticsearch indexing, ingestion workers, and legal hold logic share a single process — the primary decomposition target.
  • Establish ADR (Architecture Decision Record) practice — all future decomposition decisions must have an ADR.
  • Freeze all new feature work in the monolith ingestion layer for 90 days.
✓ OTel traces across 100% of monolith ✓ 6 domain teams formed ✓ 18PB on Intelligent-Tiering ✓ COGS baseline established: $24,800/day
W1
Wave 1

Ingestion Decoupling — "Sever the Intake Path"

Extract the message ingestion workers from the monolith. Introduce Kafka (MSK) as the durable message bus. The monolith continues to consume from Kafka — the Strangler Fig replaces the producers first.
✓ COMPLETE Q3–Q4 2025 · 6 months
Strategic Focus (Strangler Fig)
  • Deploy new Ingestion Microservice (T1) alongside monolith — initially dark-released at 0% traffic.
  • Introduce Amazon MSK (Kafka) as the durable event bus. Monolith becomes a Kafka consumer, not producer.
  • Protocol adapters (SMTP/IMAP/Graph) migrated to standalone ECS Fargate connector services.
  • 100/0 → 50/50 → 0/100 traffic shift using weighted routing in API Gateway. Zero downtime cutover via parallel-write period.
  • Monolith's ingestion code paths deprecated. Database writes to legacy ingestion tables halted.
60PB Data Plane Levers
  • New ingestion microservice writes directly to structured S3 prefix paths (/tenant/{id}/year/month/day/) enabling precise lifecycle policy targeting.
  • Separate S3 buckets per data classification (hot/warm/archive) — removes all objects from the combined monolith bucket.
  • S3 Replication to secondary region for compliance: replication costs offset by decommissioning monolith EBS volumes.
  • Deduplication engine reduces duplicate message storage by estimated 8–12% of hot-tier footprint.
Tech Debt Strategy
  • Eliminate 3 legacy vendor SDKs (custom SMTP library, bespoke IMAP parser) — replace with standardized Apache James SMTP server.
  • Remove 47,000 lines of ingestion code from monolith codebase (ArchUnit-verified zero remaining coupling).
  • Kill the legacy Oracle-backed message queue table (5M rows/day) — replaced by Kafka with 7-day retention.
  • Payoff: 6ms average indexing lag caused by shared DB mutex lock — eliminated.
✓ Ingestion on Kafka — zero monolith coupling ✓ 3PB hot zone restructured per tenant ✓ Legacy Oracle queue decommissioned ✓ COGS reduced $2,800/day (ingest tier)
W2
Wave 2

Search Plane Extraction — "Decouple the Index Brain" 🔵 IN PROGRESS

Extract Elasticsearch/OpenSearch indexing from the monolith. Separate the control plane (API Gateway, tenant config) from the search execution layer. This is the highest-risk wave — search downtime means compliance failure.
● IN PROGRESS Q1–Q2 2026 · Target: June 2026
Strategic Focus (Strangler Fig)
  • Migrate from self-hosted Elasticsearch on EC2 to OpenSearch Serverless collections — eliminates cluster management toil and scales to 60PB metadata indexing automatically.
  • New Search Microservice (T2) exposed via GraphQL API. E-Discovery team (T4) migrated to consume new search.v2 API — monolith search paths deprecated last.
  • Dual-write period: new indexing pipeline and monolith Elasticsearch run in parallel for 60 days. Index consistency checker job runs continuously. Cutover only when delta <0.001%.
  • Control Plane (T5) API Gateway deployed as the single entry point for all external calls — monolith's REST endpoints proxied through it, enabling future deprecation without client changes.
60PB Data Plane Levers
  • Separate control plane from data plane: API calls go to T5 API Gateway; bulk data reads go directly to S3 PresignedURL — eliminating expensive API-mediated data transfers.
  • OpenSearch Serverless OCUs (OpenSearch Compute Units) auto-scale with query volume — no over-provisioned clusters idle at 3 AM.
  • Index only metadata + extracted text, not raw message blobs. Raw storage stays in S3 tiered storage — search index 40x smaller than full object footprint.
  • Implement S3 Select for cold-tier result expansion — avoids full Glacier restores for e-discovery snippets.
  • Elasticsearch EC2 cluster decommission saves estimated $9,800/mo in compute.
Tech Debt Strategy
  • Priority: Eliminate the synchronous indexing code path in the monolith. Current design blocks ingestion workers waiting for Elasticsearch ACK — a critical latency debt costing 80–200ms per message.
  • Replace all raw Elasticsearch 6.x Java client calls (deprecated) with modern OpenSearch Java client — eliminates 23 CVEs in current dependency tree.
  • E-Discovery search had 4 separate query builders — consolidate into 1 unified query DSL layer owned by T2.
  • Monolith index-rebuild job (weekly, 18-hour run) deprecated — replaced by event-driven incremental reindex pipeline on Kafka.
🔵 Target: OpenSearch Serverless live July 2026 🔵 T5 API Gateway intercepting 100% of external calls 🔵 Target: −$9,800/mo compute (EC2 cluster kill) 🔵 Synchronous ingest-index coupling eliminated
W3
Wave 3

Data Plane & FinOps Optimization — "The 60PB Restructure"

Full separation of the control and data planes. Aggressive Glacier lifecycle migration to optimize the 30PB cold/deep-archive footprint. Metadata cold-store migration to DynamoDB. Target: 34% total COGS reduction.
PLANNED Q3 2026 Q3–Q4 2026 · 6 months
Strategic Focus
  • Complete control/data plane separation: all S3 data accessed via signed URLs from the data plane service — control plane (T5) never proxies object bytes.
  • E-Discovery (T4) microservice fully decoupled from monolith — owns its own Postgres metadata store, independent deployment pipeline.
  • Multi-tenancy compliance enforcement moved from monolith to T5 Authorization Service — per-request policy evaluation with OPA (Open Policy Agent).
  • Monolith is now a "thin shell" — only auth session management and legacy UI rendering remain.
60PB Data Plane Levers
  • Tier 4: Glacier Migration — transition 21PB of 730d-7yr data from S3-IA to Glacier Flexible Retrieval. Saves ~$0.013/GB/mo × 21,000,000 GB = $273,000/mo.
  • Tier 5: Deep Archive — migrate 7yr+ data (9PB) to Glacier Deep Archive at $0.00099/GB/mo. From $0.023/GB/mo S3 standard cost = $2M+/yr savings on this tier alone.
  • DynamoDB cold-store for message metadata (from/to/subject, dates, custodians) — eliminates expensive S3 metadata GET at e-discovery query time.
  • AWS Savings Plans committed for S3 and EC2 — 6-month usage pattern established in W0-W2 enables accurate commitment. Est. 26% savings on compute.
  • S3 Batch Operations to retroactively tag and reclassify existing objects at petabyte scale.
Tech Debt Strategy
  • Decommission monolith's own retention policy engine — replaced by T3's dedicated Lifecycle Policy Service with per-tenant, per-regulation configurability.
  • Remove shared-database coupling between E-Discovery (T4) and Control Plane (T5) — last remaining cross-domain database join eliminated.
  • Replace legacy RDS instance (PostgreSQL 11 EOL) hosting monolith metadata with Aurora Serverless v2 — auto-scaling, 0 idle cost, IAM auth only.
  • Eliminate 12 scheduled batch jobs (nightly, weekly) — replaced by event-driven Lambda functions. Reduces EC2 fleet by 40%.
Target: $16,400/day COGS (−34% from baseline) Target: 30PB migrated to Glacier tiers Target: Control/Data plane 100% separated Target: Monolith <5 services remaining
W4
Wave 4

Legacy Deprecation & Event Mesh — "Kill the Monolith"

Full retirement of the legacy monolith codebase. Deploy Amazon EventBridge as the enterprise event mesh. All 6 domain services communicate exclusively through event contracts. Strangler Fig pattern 100% complete.
PLANNED Q1 2027 Q1–Q2 2027 · 6 months
Strategic Focus
  • Final monolith services (legacy UI, session management) migrated to dedicated microservices. Monolith EC2 fleet decomm.
  • Amazon EventBridge replaces all remaining point-to-point service dependencies — full async event mesh topology.
  • Service mesh (Istio) mTLS enforcement across all inter-service communication. Zero trust networking complete.
  • Chaos engineering baseline established — chaos experiments validate resilience of all 6 domain services independently.
  • Runbook retirement: all monolith-era operational runbooks archived. New domain-specific playbooks published in Backstage.
60PB Data Plane — Final State
  • S3 Object Lock COMPLIANCE mode verified across 100% of the 60PB footprint — immutability enforcement audited.
  • Cross-region replication (us-east-1 → us-west-2) automated via Storage Lifecycle team's IaC — no manual steps for DR.
  • Final storage state: 5% Hot / 15% Warm / 30% Intelligent-Tiering / 35% Glacier FR / 15% Glacier Deep Archive.
  • Data gravity plane is a first-class system — all storage metrics published as CloudWatch custom metrics and surfaced in the Strategy Hub dashboard.
Tech Debt — Final Payoff
  • Monolith codebase archived (read-only Git branch) — no production traffic, no on-call rotation, no deployments.
  • Total CVEs addressed: 180+ critical/high vulnerabilities eliminated from removed components.
  • Engineering cognitive load: from 1.2M LOC monolith to 6 services averaging 40K–80K LOC each.
  • DORA metrics target: Elite performer across all 4 metrics for all 6 teams.
  • On-call load: 60% reduction in incident volume. Each team owns its domain's incidents end-to-end.
Target: Monolith 100% decommissioned Target: EventBridge event mesh live Target: DORA Elite on all 6 teams Target: $14,200/day COGS (final steady-state)

Technical Debt Prioritization Matrix

🔴Synchronous Ingest-Index Lock
Ingestion workers block on Elasticsearch ACK before marking messages as durably received. At 4.2TB/day throughput, this mutex creates cascading back-pressure during peak hours.
CRITICAL — Wave 2
FinOps Impact: +$3,200/day in over-provisioned Kafka broker capacity. Search Latency: +80–200ms P99.
🟠Shared E-Discovery Database
Legal hold queries and control-plane tenant queries share the same RDS instance. E-discovery bulk exports cause read contention degrading API response times for all tenants.
HIGH — Wave 3
FinOps Impact: Requires oversized RDS r6g.4xlarge ($1,800/mo). Cross-team deployment coupling 2× per quarter.
🟠Weekly Full Index Rebuild Job
A 18-hour nightly batch job rebuilds Elasticsearch indices from scratch due to lack of event-sourced incremental updates. Consumes 40% of search cluster compute for 75% of the week.
HIGH — Wave 2
FinOps Impact: $8,400/mo idle compute. During rebuild, search P99 degrades 3–5×.
🟡Untiered Cold Object Storage
21PB of data >2 years old remains in S3 Standard-IA, not yet transitioned to Glacier, due to missing lifecycle policies on pre-tagging era objects.
MEDIUM — Wave 3
FinOps Impact: $273,000/mo overspend vs. Glacier Flexible Retrieval pricing for this tier.
