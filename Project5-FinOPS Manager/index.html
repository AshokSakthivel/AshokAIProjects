§00 — Challenge

Cloud FinOps
From Cost Centre to Value Engine

Mimecast's AWS bill is growing faster than revenue. Without real-time cost attribution, rightsizing discipline, and a culture of cost ownership, engineering teams build without financial accountability — turning every sprint into invisible spend. This FinOps practice changes that.

Ashok Sakthivel · Sr. Director of Engineering
Mimecast · Bangalore Hub
April – June 2026
Target: 35% Cloud Savings
⚠ Current State
  • Cloud spend growing 28% YoY with no unit economics baseline
  • Resource tagging coverage below 40% — no cost attribution
  • Dev/test environments running 24×7 with zero auto-shutdown
  • 70% On-Demand pricing — Reserved Instance & Savings Plans underutilised
  • Storage costs invisible; cold data in hot S3 Standard
⚡ Root Causes
  • No Cost Ownership: Engineers optimise features, not bills
  • Tag Debt: 60% of resources have no cost-allocation tags
  • Zombie Resources: Unattached EBS volumes, idle EC2s, orphaned snapshots
  • No Showback: Teams never see their cloud bill impact
  • Missing Anomaly Alerts: Spend spikes go undetected for weeks
✅ Target State
  • Every team has real-time cost visibility dashboards (showback → chargeback)
  • 100% resource tagging with enforced AWS Tag Policies
  • Dev/test auto-scheduled — zero idle spend after 18:00 local
  • RI + Savings Plans coverage ≥80% for steady-state workloads
  • Unit cost ($/GB archived) reduced 40% by end of Q2 2026

§02 — 90-Day Roadmap

Four-Phase FinOps Transformation

1
Phase 1 — Visibility Days 0–30
Foundation
›

Deliverables

  • AWS Cost Explorer + Cost & Usage Report (CUR) setup
  • Enforce Tag Policy: env, team, product, cost-centre, owner
  • Showback dashboards per team in AWS QuickSight
  • Baseline unit cost ($/GB, $/user, $/request) established
  • Anomaly detection rules live in AWS Cost Anomaly Detection

Success Criteria

  • 100% resources tagged within 30 days
  • Every product team has a spend dashboard bookmarked
  • Anomaly alerts fired at least once (proves the system works)
  • Zombie resource audit complete — delete list approved
  • FinOps Slack channel #cloud-cost-ops live with daily digest
2
Phase 2 — Optimisation Days 31–60
Savings
›

Deliverables

  • EC2 + RDS rightsizing — AWS Compute Optimizer recommendations actioned
  • Reserved Instances purchased for steady-state DB fleet
  • S3 Intelligent-Tiering applied to all buckets ≥100GB
  • Dev/test auto-shutdown Lambda schedules deployed (18:00–08:00 + weekends)
  • Delete approved zombie resources — EBS, snapshots, unattached ENIs

Success Criteria

  • Dev/test idle spend reduced ≥50% in first 30 days
  • S3 storage costs drop ≥20% with tiering
  • EC2 rightsizing delivers ≥15% compute savings
  • RI coverage reaches 60% steady-state EC2
  • Zombie cleanup saves ≥$50K/month

Phase 3 — Governance Days 61–90
Policy
›

Deliverables

  • Cloud budget gates in CI/CD — PR blocked if Terraform adds >$X/month

Success Criteria

  • Budget variance ≤5% vs forecast for 2 consecutive months
  • RI + SP coverage reaches ≥80%
  • Zero untagged resources at month-end
  • All Terraform PRs include estimated monthly cost diff
  • CFO sign-off on chargeback model

  • Unit economics OKR embedded in every team's quarterly goals

  • FinOps maturity scorecard published quarterly to leadership
  • Cost-aware architecture patterns in Golden Path templates

  • Cloud savings rate ≥35% sustained for quarter
  • Unit cost ($/GB) reduced 40% vs baseline

§05 — Cost Strategy

FinOps Playbook — Five Levers

Lever 1

Rightsizing

AWS Compute Optimizer + instance family downsizing. Target: 15–20% EC2/RDS cost reduction. Monthly automated scan with Slack recommendations.

Lever 2

Storage Tiering

S3 Intelligent-Tiering at PB scale. Glacier Instant for 2–7yr archive. Glacier Deep Archive ($0.00099/GB) for 7yr+ compliance. EBS gp2 → gp3 migration (-20% cost).

Lever 3

Commitment Discounts

1yr Reserved Instances for baseline RDS/EC2 fleet. Compute Savings Plans for flexible workloads. Target 80% coverage = 40–72% discount vs On-Demand.

Lever 4

Scheduling & Auto-Shutdown

Lambda-powered dev/test stop schedules. Instance Scheduler for 12hr overnight + 16hr weekends. 5 environments × $30K/month → saves ~$180K/year.

Lever 5

Architecture Efficiency

Migrate batch workloads to Spot Instances (70–90% savings). Graviton3 adoption for compatible services. Serverless for event-driven low-traffic APIs.

Governance

Cost Culture

FinOps Guild: 1 champion per squad. Cloud cost OKR in every team's quarterly goals. Weekly #cloud-cost-ops digest. Monthly leadership FinOps review with CFO.

§07 — Anti-Pattern Registry

FinOps Failure Modes

Anti-PatternSeverityRoot CauseMitigation
📢 No Cost Ownership
Engineers treat cloud cost as someone else's problem — finance's, not engineering's
 High No showback; no chargeback; cloud bill abstracted from teams
Chargeback + weekly team digest. Cost OKR in sprint goals. FinOps champion in each squad. Public savings leaderboard.
🏷 Tag Debt Spiral
Resources created without tags — cost attribution impossible; savings targets unverifiable
 High No IaC enforcement; manual resource creation bypassed tagging
AWS SCP blocks untagged resource creation. Terraform modules enforce tags. Weekly non-compliance report sent to team lead.
🧟 Zombie Resource Accumulation
Idle EC2s, unattached EBS, orphaned snapshots and unused load balancers accumulate silently
 Medium No automated cleanup; engineers fear deleting shared resources
AWS Trusted Advisor + Cost Anomaly Detection. Monthly zombie sweep with auto-stop (not delete) after 14-day idle. Owner notification workflow.
⏰ Over-Provisioning Habit
Teams request largest instance sizes "just in case" — rightsizing never revisited post-launch
 Medium No post-launch cost review; performance reviews don't include cost
AWS Compute Optimizer monthly runs. Rightsizing recommendations pushed to team Jira backlog. Require cost justification for m5.xlarge+.
💤 Always-On Dev/Test
Non-production environments run 24×7, consuming production-equivalent compute budgets
 Medium No scheduling policy; devs afraid environments won't start cleanly
Instance Scheduler deployed day 1. Opt-out mechanism for on-call environments. Start/stop Lambda tested for all 5 dev environments before policy enforcement.
📉 Vanity Savings Metrics
Savings reported in absolute $ without normalising for spend growth — appears efficient while wasting more
 Low Finance reporting focuses on absolute credits; unit economics ignored
Always report savings rate % alongside absolute $. Unit cost ($/GB, $/user) as primary FinOps KPI. Baseline locked at Q1 2026 spend.
