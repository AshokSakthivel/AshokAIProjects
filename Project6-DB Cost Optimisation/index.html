§00 — Challenge

Database Fleet
Slash Cost, Preserve Performance

Mimecast's database fleet — 60PB of email archive data plus transactional DB clusters — is the single largest cost driver in the AWS bill. Oversized instances, cold data sitting in hot storage, and manual operations inflate cost by an estimated 40–50% above optimal. This initiative changes that.

Ashok Sakthivel · Sr. Director of Engineering
Mimecast · Global Engineering
April – June 2026
Target: 40% DB Cost Reduction
⚠ Current State
  • $4.2M/year DB spend — 45% of total AWS bill
  • 30% of RDS instances oversized (p50 CPU < 15%)
  • 60PB archive: 90% stored in S3 Standard — should be Glacier Deep Archive
  • No connection pooling — each Lambda creates direct DB connections
  • Snapshots retained indefinitely — 8TB of weekly snapshot clutter
  • Query P99 latency degraded by unindexed eDiscovery queries on archive
⚡ Root Causes
  • Fear-Based Sizing: "Larger = safer" mentality with no post-launch review
  • Cold Data in Hot Storage: No lifecycle policy on archive S3 buckets
  • No Query Governor: Expensive full-table scans not gated at app layer
  • Snapshot Sprawl: Manual snapshots never deleted; automated retention uncapped
  • No RDS Proxy: Connection storm on eviction events spikes DB CPU
✅ Target State
  • DB monthly spend reduced ≥40% — $1.68M/year savings
  • 60PB archive: S3 Intelligent-Tiering + Glacier Deep Archive for 7yr+ data
  • All RDS clusters rightsized — CPU p50 ≥40% utilisation
  • RDS Proxy deployed — connection pooling for all Lambda-backed services
  • Automated snapshot lifecycle: 7-day transactional, 30-day archive, delete rest
  • eDiscovery queries served by Athena — zero production DB query load

§05 — Optimisation Levers

Six-Lever Database Cost Strategy

Lever 1

Graviton3 Migration

Migrate Aurora PostgreSQL to Graviton3 (db.r6g / db.t4g) for -35% instance cost with equivalent performance. Validated by AWS Graviton migration guide on Mimecast workload profiles.

Lever 2

Glacier Deep Archive

7yr+ compliance email archive at $0.00099/GB vs $0.023/GB in S3 Standard — 95% cost reduction on cold archive. Per-tenant S3 Lifecycle rules auto-tier data by ingestion age.

Lever 3

RDS Proxy + Connection Pools

RDS Proxy eliminates Lambda-RDS connection storms. Reduces DB CPU by 15–20% during eviction bursts. Required before any further Lambda function scale-up.

Lever 4

Athena for eDiscovery

Route all reporting and eDiscovery queries to Athena + Glue catalog over S3. Eliminates production DB query load. Athena cost model: $5/TB scanned vs RDS IOPS charges.

Lever 5

Aurora Serverless v2

Replace fixed-size dev/test Aurora clusters with Serverless v2 (0.5 ACU auto-pause). Saves ~$15K/month on 8 non-production environments that are idle 14+ hours/day.

Lever 6

Snapshot Lifecycle Automation

Lambda-triggered snapshot cleanup: 7-day retention for transactional, 30-day for archive, delete all else. Eliminates 8TB of redundant snapshots costing $160/TB/month.

§07 — Anti-Pattern Registry

DB Optimisation Failure Modes

Anti-PatternSeverityRoot CauseMitigation
📦 Cold Data in Hot Storage
7-year email archive sitting in S3 Standard at $0.023/GB — 23× the cost of Glacier Deep Archive
 High No lifecycle policy applied at bucket creation; "set and forget" approach
S3 Lifecycle rules via CDK. Intelligent-Tiering for 0–2yr, Glacier IR for 2–7yr, Deep Archive for 7yr+. Legal hold tag prevents aggressive tiering on litigation data.
⚡ Lambda-RDS Connection Storm
Lambda functions open direct DB connections — 1000 concurrent invocations = 1000 DB connections, spiking CPU and causing connection refused errors
 High No RDS Proxy; Lambda treated as stateful service despite ephemeral nature
RDS Proxy mandatory for all Lambda-RDS pairs. Connection limit config. Alarm when DB connections >80% of max_connections.
📊 eDiscovery on Production DB
Legal/compliance teams run full-table reporting queries directly on production Aurora — causes IOPS spikes, query latency degradation, and unpredictable cost
 High No dedicated analytics tier; Athena not deployed; ad-hoc SQL access given to legal team
Athena + Glue over S3 archive. Revoke direct DB access for non-engineering roles. Athena workgroup with query cost limits per user.
💾 Snapshot Sprawl
Manual and automated snapshots accumulating without retention policy — 8TB of redundant snapshots at $0.05/GB/month
 Medium No AWS Backup policy; default "never delete" manual snapshot behaviour
AWS Backup plans: 7-day for transactional, 30-day for archive. Lambda to auto-delete snapshots exceeding policy. Monthly cost report to engineering leads.
💤 Always-On Dev DB Clusters
8 non-production Aurora clusters running 24×7 at full production capacity — idle 14+ hours/day
 Medium No Aurora Serverless v2 adoption; engineers assume dedicated cluster = better performance
Aurora Serverless v2 for all non-production clusters. Auto-pause after 30min idle. Capacity range: 0.5–8 ACU (never idle-costs full r5.4xlarge equivalent).
📉 Fear-Based Instance Sizing
Engineers refuse rightsizing recommendations citing "Black Friday risk" — applies to non-seasonal workloads year-round
 Low No data-driven sizing culture; one high-traffic incident drove over-provisioning for all clusters
30-day monitoring window pre-downsize. Scheduled scaling for known peak events. Performance Insights baseline shared with team lead before change approval.
